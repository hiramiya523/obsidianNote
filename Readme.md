# 📖 このVaultについて

技術メモ・ナレッジベースを管理するObsidian Vaultです。

## 使い方

1. **[[Index|🏠 Index]]** から始めてください
2. 各ドキュメント間は内部リンク（`[[ページ名]]`）で移動できます
3. タグやグラフビューで関連情報を探索できます
4. 検索機能（`Ctrl+Shift+F`）で全文検索が可能です
5. バックリンクパネルで、どのページからリンクされているか確認できます

## Obsidianの機能

このVaultでは以下のObsidian機能を活用しています：

| 機能 | 説明 | 例 |
|------|------|-----|
| **内部リンク** | ドキュメント間のナビゲーション | `[[ページ名]]` または `[[ページ名\|表示テキスト]]` |
| **フロントマター** | メタデータ管理（YAML形式） | title, tags, aliases, created |
| **Callout** | 重要情報の視覚的強調 | `> [!warning]` `> [!tip]` `> [!info]` 等 |
| **タグ** | カテゴリ分類 | `#docker` `#linux` `#frontend` |
| **バックリンク** | リンク元のページを表示 | サイドバーで確認可能 |
| **グラフビュー** | ページ間の関係を可視化 | `Ctrl+G` で開く |
| **検索** | 全文検索 | `Ctrl+Shift+F` |
| **エイリアス** | ページの別名を設定 | フロントマターの`aliases`で設定 |

## カテゴリ

| フォルダ | 内容 | 主要タグ |
|---------|------|---------|
| `Docker/` | Docker, Fluent Bit, ログ管理 | `#docker` `#logging` `#fluent-bit` |
| `Linux/` | Linuxコマンド, サーバー設定 | `#linux` `#server` `#security` |
| `Windows/` | Hyper-V, VM構築 | `#windows` `#hyper-v` `#vm` |
| `Database/` | SQL, パフォーマンス最適化 | `#sql` `#database` `#mysql` |
| `Frontend/` | SvelteKit, Svelte, TypeScript | `#sveltekit` `#svelte` `#frontend` |
| `Projects/` | プロジェクト固有の計画 | `#project` `#migration` |

## Calloutの凡例

このVaultで使用しているCalloutの種類：

> [!info] 情報
> 補足説明や参考情報を提供します。

> [!tip] ヒント
> 推奨事項やベストプラクティスを示します。

> [!success] 成功・メリット
> 成功例やメリットを説明します。

> [!warning] 注意
> 注意すべき点や制限事項を示します。

> [!danger] 危険
> 重大な注意点やセキュリティリスクを示します。

> [!bug] バグ・問題
> 既知の問題やトラブルシューティング情報を示します。

> [!abstract] 概要
> ドキュメントの概要や要約を示します。

> [!example] 例
> コード例や使用例を示します。

> [!question] 質問
> よくある質問や疑問点を示します。

> [!note] 注記
> 補足的な注記や参考情報を示します。

> [!failure] デメリット
> デメリットや制約を示します。

## ナビゲーションのヒント

### グラフビューで探索
- `Ctrl+G` でグラフビューを開く
- ノードをクリックしてページを開く
- ズームやパンで全体像を把握

### バックリンクで関連ページを発見
- サイドバーのバックリンクパネルを開く
- 現在のページにリンクしているページを確認
- 関連する情報を効率的に発見

### タグでフィルタリング
- `#docker` でDocker関連の全ページを表示
- `#linux` でLinux関連の全ページを表示
- タグパネルから直接ページにアクセス

### 検索の活用
- `Ctrl+Shift+F` で全文検索
- 正規表現検索も可能
- ファイル名検索: `Ctrl+O`
